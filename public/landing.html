A wallet stack for agent builders.

Use

Ask for the outcome. bunOS handles the rails.

The interface is intentionally split: normal users use wallet actions, power users type commands, and agents connect over MCP.

Payments Pay an X handle.

Send USDC to onboarded users or create a claimable payment for someone who has not connected yet.

Tokens Swap supported Arc assets.

Route USDC, EURC, cirBTC, WETH, and other supported tokens through the same agent surface.

Bridge Move value across rails.

Use Arc and Base Sepolia for cross-chain movement while keeping receipts and execution status visible.

Perps Trade from a prompt.

Create ArcPerps testnet positions from natural language without pretending failures succeeded.

Safety

No shared backend signer for user funds.

This is the part worth being blunt about. A slick agent wallet becomes dangerous fast if every user rides one hidden key. bunOS scopes tools to the user's wallet context.

Signing model

User-wallet scoped execution.

API keys bind MCP clients to the same wallet created after X login. Protocol admin keys are not used as user spending keys.

Receipts Every serious action needs a status trail.

Pending, submitted, failed, or completed. No fake success screens.

Risk Large actions can be policy gated.

Payments, swaps, bridges, and perps can be checked before execution.

MCP

Connect any agent to the same wallet.

The MCP guide and API Keys page now use the canonical domain. No one should be copy-pasting Railway URLs into clients.

One URL. User scoped tools.

Create an API key after signing in with X, paste the MCP config into Claude, Cursor, Windsurf, or another MCP client, then call the same wallet tools the terminal uses.
