How it works

Four steps, no address juggling.

The app is built for a normal user: connect, fund, ask, and verify.

Step Connect X

OAuth links the account. Local demo mode can still create a test handle while X is being configured.

Step Get a wallet

bunOS creates a Circle wallet for that user and syncs token balances by rail.

Step Use buttons or chat

Dashboard actions and terminal commands call the same backend tools.

Step Check receipts

Completed operations show status, route detail, and transaction receipt links when execution returns them.

Safety model

Clear about what is real.

No fake success screens. If a route cannot execute, bunOS should say why instead of pretending.

User funds do not ride a shared backend wallet.

User operations are routed through that user's Circle wallet. The protocol admin key is only for protocol-level setup and oracle/admin duties.

Testnet by default Built for safe demos

Arc testnet execution lets users try payments, swaps, bridges, and perps without mainnet funds.

Visible status Receipts over vibes

Actions show pending, submitted, failed, or completed states so the user knows what happened.
